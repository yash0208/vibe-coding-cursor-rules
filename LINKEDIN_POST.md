🚀 Stop vibe coding blind.

AI agents (Cursor, Claude Code, Copilot) are incredible — but they guess when you don't give them context. Wrong stack. Broken navigation. Tables that don't match your app. Hours of undo.

I packaged the fix into a free, public repo:

**6 documents. Write them BEFORE any code.**

1️⃣ PRD — what you're building & for whom  
2️⃣ TRD — stack, APIs, env vars (locked in)  
3️⃣ App Flow — every page, click, redirect  
4️⃣ UI/UX Brief — colors, fonts, vibe (not generic gray boxes)  
5️⃣ Backend Schema — tables, auth, RLS  
6️⃣ Implementation Plan — phased build order with done criteria  

Think of it as the Day 1 briefing you'd give a senior dev — except your "dev" is an AI that never sleeps.

**How I use it:**  
→ Fill the 6 templates (10–20 min with ChatGPT/Claude)  
→ Drop the Cursor rules into `.cursor/rules/`  
→ Start every session: *"Use my 6 docs as source of truth"*  

Result: fewer hallucinations, less back-and-forth, cleaner architecture, faster shipping.
